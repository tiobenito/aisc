Guide 02

How to Download and Install a Skill

Get a shared skill onto your machine and into Claude Code so it's available in every session.

What's a Skill?

A skill is a reusable instruction file that teaches Claude Code how to do a specific task. Someone builds it once, and anyone can install it. Skills live in ~/.claude/skills/ on your computer.

Prerequisites

  • Claude Code installed and working — see Guide 01
  • A terminal you're comfortable typing commands into
  • A skill to install — either a GitHub link or a zip file from your instructor

Option A: Install from GitHub (most common)

If your instructor shares a GitHub link, clone it directly in your terminal:

zsh
$ git clone https://github.com/[username]/[repo-name]

Example — the AISC skill-builder:

zsh
$ git clone https://github.com/tiobenito/aisc

This downloads the whole repo to your current folder. You'll see a new folder appear.

Tip

Not sure which folder you're in? Type pwd in your terminal to see.

Option B: Download as a zip

If you don't want to use git:

1

Go to the GitHub repo in your browser

Open the link your instructor shared.

2

Click the green Code button → Download ZIP

Look for the green Code button near the top of the page. Click it, then choose Download ZIP from the dropdown.

3

Unzip the file

Double-click the downloaded .zip file to extract it — you'll get a folder on your computer, usually in Downloads.

Install the Skill

Once you have the folder, move it into Claude's skills directory. Follow these steps in order.

1

Find the SKILL.md file

Navigate into the folder you downloaded. Somewhere inside is a SKILL.md file — that's the skill. It might be at the top level, or inside a subfolder.

Example: if you cloned the aisc repo, the skill-builder is at aisc/skill-builder/SKILL.md.

2

Move the skill folder into ~/.claude/skills/

The folder containing SKILL.md is what you're installing. Copy it into Claude's skills directory:

zsh
$ cp -r path/to/skill-name ~/.claude/skills/skill-name

Concrete example:

zsh
$ cp -r aisc/skill-builder ~/.claude/skills/skill-builder

If ~/.claude/skills/ doesn't exist yet:

zsh
$ mkdir -p ~/.claude/skills/
3

Verify the structure

zsh
$ ls ~/.claude/skills/skill-builder/ SKILL.md

You should see SKILL.md inside. If it's nested one more level deep, move it up — Claude won't find it otherwise.

Terminal showing SKILL.md inside the skill folder

4

Test it

Start Claude Code in any project and try a phrase that should trigger the skill. Check the SKILL.md description field for the trigger phrases to use.

If Claude responds with the skill's behavior, you're good.

Troubleshooting

Skill doesn't trigger

Make sure the folder is directly inside ~/.claude/skills/, not nested an extra level. The path should be ~/.claude/skills/[skill-name]/SKILL.md.

"No skill found"

The file must be named exactly SKILL.md — capital S, capital MD. Rename it if needed.

Skill runs but behaves wrong

Read the SKILL.md file to see if it has prerequisites or setup instructions you missed.

Permission error on copy

Try sudo cp -r ... and enter your password when prompted.
