Guide 05

How to Recover When Things Break

How to undo mistakes — starting with asking Claude directly, and falling back to terminal commands when needed.

Prerequisites

  • Claude Code installed and running — see Guide 01
  • A project with at least one Git commit

Before You Panic

Almost nothing is truly lost. Claude Code works with Git underneath, which means there are save points to go back to. Take a breath and pick the right approach below.

The Easy Way: Ask Claude

If you're already in a Claude Code session, just ask. Claude can run Git commands for you — you don't need to know the exact command.

1

Undo the last set of changes

Type this directly into Claude Code:

claude
Undo the changes you just made
2

Revert a specific file

claude
Revert App.tsx to how it was before
3

Go back to a specific point

claude
Can you show me recent commits? I want to go back to before we added authentication.
4

When the whole thing is a mess

claude
Something went wrong and I want to start fresh from the last commit. What should we do?

Claude will figure out the right Git commands and run them. If it's not sure, it'll ask you before doing anything destructive.

Tip

Claude keeps a running history of the changes it made in a session. If something went wrong, telling it "that broke something, undo the last change" is often enough.

The Terminal Way (if you're outside Claude)

If you're not in a Claude session or want to do it yourself, here are the commands.

1

Undo everything — changes not committed yet

This saves your changes in a hidden stash and reverts your files to the last commit. Your work isn't gone — it's tucked away.

zsh
$ git stash

To bring it back:

zsh
$ git stash pop
2

Undo changes to one specific file

This resets that one file to the last commit. Everything else stays the same.

zsh
$ git checkout -- src/App.tsx
3

Undo a bad commit

This undoes your last commit but keeps your file changes. Fix things and commit again.

zsh
$ git reset --soft HEAD~1
4

Go back to an older commit

First, find the short hash (like abc1234) from before things broke:

zsh
$ git log --oneline
⬚ Screenshot or screen recording

Terminal showing git log with commit hashes

Then create a recovery branch from that point:

zsh
$ git checkout -b recovery abc1234

The Golden Rule

Commit often. The more commits you have, the more save points you can go back to. Commit every time something works — even small things. If you do that, recovery is always easy.

Troubleshooting

Not sure what happened

Ask Claude "show me what changed since the last commit" or run git status in the terminal.

Claude isn't sure what to undo

Be specific: "undo the changes to App.tsx" rather than "fix this."

Stash seems empty

Check git stash list to see all your stashes.
