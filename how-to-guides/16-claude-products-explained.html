Guide 16

Claude Products Explained

The difference between Claude Chat, Claude Code CLI, Claude Code Desktop, and Claude Cowork — and why this course uses the CLI.

Why This Is Confusing

Anthropic has one AI (Claude) but multiple ways to use it. Students often ask: "Why are we using the terminal? Can't I just use the desktop app?" or "What's Cowork?"

Think of Claude as the brain. The products are different bodies you can put it in — each one gives it different abilities.

1. Claude Chat

The one most people know. You type, Claude responds. Available at claude.ai in your browser, on your phone, or in the Chat tab of the desktop app.

What it does Answer questions, brainstorm, write, analyze, search the web, create artifacts (charts, docs, code previews), upload files, connect to Gmail/Calendar/Notion/Slack, voice conversations
What it can't do Read/write files on your computer, run terminal commands, work on a full codebase, do multi-step tasks autonomously
Best for Everyone. No technical skills needed. Quick questions, writing, research, brainstorming
Cost Free plan available. Pro ($20/mo) for higher limits and Opus

2. Claude Code CLI (Terminal)

This is what we use in this course. You open your terminal, navigate to a project folder, type claude, and start building. Claude reads your entire project, edits files, runs commands, and manages git — all from the terminal.

What it does Read your entire codebase, edit multiple files, run any terminal command (npm, git, tests), create commits/branches/PRs, remember project context (CLAUDE.md), install skills, connect to external tools (GitHub, Notion), run on remote servers, work in CI/CD pipelines
What it can't do Show visual diffs (text only), preview your app in a browser, control your screen — no GUI, it's all text
Best for Developers and power users who want maximum control. People comfortable with (or learning) the terminal
Cost Pro ($20/mo) or higher. Not available on the free plan

3. Claude Code Desktop (Code Tab)

The same Claude Code engine, but wrapped in a graphical interface. Click the Code tab in the desktop app instead of opening your terminal.

Everything the CLI does, plus Visual diff review (side-by-side file changes), live app preview (embedded browser), computer use (Claude controls your screen), GitHub PR monitoring with auto-fix, multiple parallel sessions, point-and-click connector setup
What it can't do Some advanced CLI features (piping, scripting). Computer use requires Pro or Max plan
Best for Developers who want Claude Code power with visual feedback. People who prefer clicking over typing commands
Cost Same as CLI — Pro ($20/mo) or higher. Not available on the free plan

4. Claude Cowork (Cowork Tab)

The newest product. Designed for non-coding knowledge work. You give Claude a goal, it breaks it into steps, and works autonomously while you do other things.

What it does Read/write files on your computer, create finished deliverables (Excel, PowerPoint, docs), research and synthesize information, organize files, control your screen, Dispatch (send tasks from your phone), scheduled recurring tasks
What it can't do Not designed for coding. Desktop app must stay open (laptop sleeps = tasks pause). Still a research preview — complex tasks succeed about 50% of the time. Burns through usage quota fast
Best for Non-technical knowledge workers — researchers, analysts, ops, legal, finance. Anyone who works with documents and data but doesn't code
Cost Pro ($20/mo) or higher. Dispatch and computer use require Pro or Max only

Why We Use Claude Code CLI in This Course

1

It's the most powerful

The CLI gives you full control — every terminal command, every file operation, every git workflow. Nothing is hidden behind a GUI.

2

It teaches you real skills

Learning the terminal, git, and command-line tools carries over to any development environment. The desktop app abstracts these away — convenient, but you don't learn as much.

3

It works everywhere

The CLI runs on any machine with a terminal — your laptop, a remote server, a CI pipeline. The desktop app only works on your local machine with the app open.

Note

The desktop Code tab is great once you're comfortable. But starting with the CLI means you understand what's actually happening underneath.

Quick Decision Guide

"I just want to ask Claude a question" Chat
"I want to build an app" Code CLI (this course) or Code Desktop
"I want visual diffs and app preview" Code Desktop
"I want Claude to research and make me a document" Cowork
"I want Claude to work while I do other things" Cowork or Code Desktop
"I'm on the free plan" Chat (only option)
