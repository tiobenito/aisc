Guide 13

How to Connect the Google Workspace CLI

Install the gws CLI and connect it to your Google accounts so you can read email, calendar, Drive, Sheets, and more — all from the terminal.

What This Is Good For

Once set up, gws gives Claude and your scripts direct access to your Google data:

  • Gmail - read emails, search your inbox, check threads
  • Calendar - list events, check your schedule
  • Drive - search files, list folders
  • Sheets - read spreadsheet data
  • Docs, Tasks, Keep, and more

You can connect multiple accounts (personal + work) and switch between them with a single env var.

1

Install gws

zsh
$ brew install googleworkspace-cli

Verify:

zsh
$ gws --version
2

Install the Google Cloud CLI

gws auth setup uses gcloud under the hood to configure your GCP project and OAuth credentials. You need it installed first:

zsh
$ brew install google-cloud-sdk

Log in:

zsh
$ gcloud auth login

This opens a browser once so Google can verify your identity. Come back to the terminal after.

3

Set up your first account

Run the setup command. It handles creating a GCP project, enabling APIs, and configuring OAuth all at once:

zsh
$ gws auth setup --login

Follow the prompts. It will:

  1. Create or select a GCP project
  2. Enable the Google Workspace APIs
  3. Open a browser to authorize your Google account
  4. Save encrypted credentials locally
Tip

Use --login --readonly if you only need to read data (no sending email, no edits). Read-only reduces the permission surface and avoids the "unverified app" warning from Google.

4

Add a second account (optional)

To connect a second account (work or personal), create a separate config directory for it and run setup pointing to that directory:

zsh
$ mkdir -p ~/.config/gws-work $ GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-work gws auth setup --login --readonly

Do the same for a personal account if needed:

zsh
$ mkdir -p ~/.config/gws-personal $ GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-personal gws auth setup --login --readonly

Each account stores its own credentials, token cache, and client config in its directory. They don't interfere with each other.

5

Switching between accounts

Set the GOOGLE_WORKSPACE_CLI_CONFIG_DIR env var before any gws command to pick which account it uses:

zsh
# Work email $ GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-work gws gmail users messages list --params '{"userId": "me", "maxResults": 5}' # Personal calendar $ GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-personal gws calendar events list --params '{"calendarId": "primary", "singleEvents": true}'
Tip

Add shell aliases to your ~/.zshrc to avoid retyping the env var:

~/.zshrc
alias gwswork='GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-work gws' alias gwspersonal='GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-personal gws'

Then use gwswork gmail ... and gwspersonal calendar ....

Useful CLI Commands

zsh
# List recent emails $ gws gmail users messages list --params '{"userId": "me", "maxResults": 10}' # Search inbox $ gws gmail users messages list --params '{"userId": "me", "q": "is:unread"}' # Today's calendar events $ gws calendar events list --params '{"calendarId": "primary", "timeMin": "2026-04-08T00:00:00Z", "timeMax": "2026-04-08T23:59:59Z", "singleEvents": true, "orderBy": "startTime"}' # List Drive files $ gws drive files list --params '{"pageSize": 20}' # Read a spreadsheet $ gws sheets spreadsheets.values get --params '{"spreadsheetId": "YOUR_SHEET_ID", "range": "Sheet1!A1:D10"}' # Check auth status $ gws auth status # Explore what a command accepts $ gws schema gmail.users.messages.list

Troubleshooting

"restricted_client" error during login

Your OAuth app hasn't been verified by Google. Use --readonly flag (less permissive = less scrutiny) or add your email as a test user in the GCP Console under OAuth consent screen.

"gws: command not found"

Close and reopen your terminal after installing.

Wrong account used

Check if GOOGLE_WORKSPACE_CLI_CONFIG_DIR is set in your shell. Run gws auth status to see which account is active.

Token expired

Run gws auth login in the relevant config directory to refresh.

Setup fails

Make sure gcloud auth login succeeded first and you have a GCP project available.
