Guide 08

How to Deploy Your Project with Vercel

Put your app on the internet so anyone can use it.

Prerequisites

  • A working project that runs locally (npm run dev works)
  • A free Vercel account (sign up at vercel.com)
  • Your code pushed to GitHub

Two Ways to Deploy

Option A: Deploy from the Vercel Dashboard (easiest)

If your code is on GitHub, this is the fastest path.

  1. Go to vercel.com and sign in
  2. Click Add New → Project
  3. Connect your GitHub account if you haven't already
  4. Find your repo and click Import
  5. Vercel detects your framework automatically. Click Deploy

That's it. You'll get a live URL in about a minute.

⬚ Screenshot or screen recording

Vercel dashboard showing the import flow

Auto-deploy on push: After connecting, every time you push to GitHub, Vercel automatically redeploys. You never have to manually deploy again.

Add environment variables from the dashboard:

  1. Go to your project settings → Environment Variables
  2. Add each variable (like INSTANT_APP_ID) with its value
  3. Redeploy for the variables to take effect

Option B: Deploy from the Terminal (Vercel CLI)

If you prefer the command line or don't have your code on GitHub yet:

1

Install the Vercel CLI

zsh
$ npm install -g vercel
2

Log in

zsh
$ vercel login

This opens your browser. Log in with your Vercel account.

3

Deploy

From your project folder:

zsh
$ vercel

Vercel will ask a few questions:

  • Set up and deploy? Yes
  • Which scope? Pick your account
  • Link to existing project? No (first time)
  • Project name? Press Enter or type a name
  • Directory? Press Enter

You'll get a URL like https://your-project.vercel.app.

⬚ Screenshot or screen recording

Terminal showing the Vercel deploy flow with the final URL

4

Add environment variables

zsh
$ vercel env add INSTANT_APP_ID

Vercel will prompt you for the value and which environments to apply it to. Pick all three (Production, Preview, Development) if unsure.

After adding env vars, redeploy to production:

zsh
$ vercel --prod

Deploy Updates

After making changes, both approaches keep you covered:

  • Dashboard: Just push to GitHub. Vercel redeploys automatically.
  • CLI: Run vercel --prod from your project folder.

The --prod flag deploys to your main URL. Without it, you get a preview URL (useful for testing changes before going live).

Troubleshooting

Build fails on Vercel but works locally

You probably have environment variables set locally that Vercel doesn't know about. Add them in the dashboard or with vercel env add.

Page shows "404" on refresh

For single-page apps (React), add a vercel.json file: {"rewrites": [{"source": "/(.*)", "destination": "/index.html"}]}

"command not found: vercel"

Close and reopen your terminal, then try again.

Changes not showing after push

Check the Deployments tab in your Vercel dashboard to see if the deploy succeeded or failed.
