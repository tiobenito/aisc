Guide 12

How to Manage API Keys and Environment Variables

Store secrets safely, keep them out of GitHub, and make them available in both local development and Vercel.

What's an Environment Variable?

It's a value your app reads from the environment it's running in - not from your code. API keys, database IDs, and other secrets live here instead of in your files.

Why: If you paste an API key directly into your code and push to GitHub, it's exposed. Anyone can find it. Environment variables keep secrets out of your codebase entirely.

The Pattern

You have two environments:

  • Local (your computer): secrets live in a .env file
  • Production (Vercel): secrets live in the Vercel dashboard

Same variable names, different places.

Local Setup

1

Create a .env file

In your project's root folder (same level as package.json), create a file called .env:

.env
VITE_INSTANT_APP_ID=your-app-id-here VITE_OPENAI_API_KEY=sk-...
Note

In Vite/React projects, variables must start with VITE_ to be accessible in your frontend code. Variables without VITE_ are backend-only.

2

Add .env to .gitignore

Make sure your .env file is never committed to GitHub. Check your .gitignore:

zsh
$ cat .gitignore

You should see .env in the list. If not, add it:

.gitignore
.env .env.local
Important

This is the most important step. A .env file committed to a public repo exposes all your secrets.

3

Create a .env.example file

So others (or future you) know what variables are needed, create a template with the variable names but no values:

.env.example
VITE_INSTANT_APP_ID= VITE_OPENAI_API_KEY=

Commit this file to GitHub - it's safe because it has no actual values.

4

Use the variable in your code

Vite / React
const appId = import.meta.env.VITE_INSTANT_APP_ID;

For Node.js / backend code:

Node.js
const apiKey = process.env.OPENAI_API_KEY;

Vercel Setup

Your .env file doesn't go to Vercel - you add variables there separately.

Option A: From the Vercel dashboard

  1. Go to your project on vercel.com
  2. Settings → Environment Variables
  3. Add each variable with its value
  4. Select all three environments (Production, Preview, Development)
  5. Save, then redeploy

Option B: From the terminal

zsh
$ vercel env add VITE_INSTANT_APP_ID

Vercel will prompt for the value and which environments to use.

After adding variables, always redeploy:

zsh
$ vercel --prod

Quick Reference

Where How to set How to read
Local (Vite/React) .env file import.meta.env.VITE_VAR_NAME
Local (Node.js) .env file process.env.VAR_NAME
Vercel Dashboard or vercel env add Same as local - Vercel injects them

Troubleshooting

Variable is undefined in Vite

Make sure it starts with VITE_. Restart the dev server after changing .env.

Works locally but not on Vercel

You haven't added the variable to Vercel yet. Check Settings → Environment Variables.

Accidentally committed .env

Remove it from Git history. Ask Claude: "I accidentally committed my .env file - how do I remove it from git history?"

Not sure what variables you need

Check your .env.example file, or look for import.meta.env and process.env in your codebase.
