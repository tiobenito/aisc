Guide 04

Git Basics Without Fear

The 5 Git commands you'll use 95% of the time. That's it.

Prerequisites

  • A project folder on your computer
  • Git installed — check by running git --version in your terminal
  • A GitHub account — sign up at github.com

Why Git?

Git saves snapshots of your code so you can go back if something breaks. Think of it like version history in Google Docs, but for your entire project.

1

git init — Start tracking a project

Run this once in a new project folder:

zsh
$ cd ~/my-project $ git init

This creates a hidden .git folder. You'll never need to touch it.

⬚ Screenshot or screen recording

Terminal showing git init output

2

git add — Stage your changes

Tell Git which files you want to save:

zsh
$ git add .

The . means "everything that changed." You can also add specific files:

zsh
$ git add src/App.tsx
3

git commit — Save a snapshot

zsh
$ git commit -m "add login page"

The message in quotes describes what you changed. Keep it short and descriptive.

Tip

Think of commits like save points in a video game. Commit often — every time something works.

⬚ Screenshot or screen recording

Terminal showing git add and git commit sequence

4

git push — Upload to GitHub

After committing, push your code to GitHub so it's backed up online:

zsh
$ git push

If it's your first push, Git will tell you what to run instead. Just copy-paste that command.

5

git pull — Download the latest changes

If someone else (or you from another computer) pushed changes:

zsh
$ git pull
6

git status — See what's going on

When you're confused, this is your best friend:

zsh
$ git status

It shows what files changed, what's staged, and what's not committed yet.

The Workflow (Every Time)

Four steps. Repeat forever.

  1. Make changes to your code
  2. git add .
  3. git commit -m "describe what you did"
  4. git push

Troubleshooting

"fatal: not a git repository"

You need to run git init first, or you're in the wrong folder.

Push fails with "rejected"

Run git pull first to get the latest changes, then try git push again.

Accidentally committed something wrong

Don't panic. See Guide 05: Recover From Mistakes for step-by-step help.
