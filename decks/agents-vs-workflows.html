Agents vs Workflows

When to define every step vs. when to let the model decide

Same Task, Two Ways

The Workflow Way
1 A trigger fires
2 Follows a pre-defined path — each step is fixed in advance
3 Produces the same output every time
Always takes the same path
The Agent Way
1 A trigger fires
2 The agent reads the situation and decides what to do next
3 Chooses which tools to call — and in what order
Path depends on what it finds

The key difference from an agent: you drive the conversation. The assistant responds to each message. It doesn't decide to go look for something on its own.

What is an Agent?

Tier 4 — has tools, decides what to do next, loops until done

“API error rate spiked to 42% at 3:42am. Handle it.”

LLM Brain
Reads + Decides
Decides which tool to call. Reads the result. Decides what to do next. Loops until the goal is done.
↻ not just one response — it iterates
→ result ←
Tools
Open-Ended Set
web_search read_file calendar browser query_db send_email
Unlike an assistant, the set isn't fixed — you give it whatever it needs.
Claude Code is an agent. System prompt = CLAUDE.md. Tools = bash, file read/write, search. It loops until the task is done — you haven't been chatting, you've been using an agent all course.

Assistant vs Agent

Both have tools. The difference is who's driving.

Tier 3 — AI Assistant
+Responds to what you ask
+Narrow scope — does specific, defined tasks
+You decide when to use it and what to ask
+Predictable — stays in its lane
"What's the status of order #1234?"
Customer support bot FAQ bot Search assistant
Tier 4 — Agent
+Pursues a goal autonomously
+Broad scope — figures out what needs doing
+Decides what to do next based on what it finds
+Multi-step — can run for hours
"Handle this customer issue end-to-end"
Claude Code · Incident response · Research agent
You drive an assistant. An agent drives itself.

"Write me a research report on EVs"
• Orchestrator breaks it into tasks: research, analyze, write
• Delegates each task to a specialist agent
• Specialists work independently, loop until their task is done
• Orchestrator assembles the final output

Trade-offs

Neither is better. They're good at different things.

Workflows Agents
Reliability
High Same path every time
Variable Depends on LLM reasoning
Flexibility
Low Fixed steps, can't adapt
High Adapts to each situation
Cost per run
Cheap Few fixed API calls
Expensive Many LLM calls per run
Debugging
Easy See each node's input/output
Harder Reasoning can be opaque
Time to build
Maybe Fast for simple flows
Maybe Fast for complex tasks
Use the simplest thing that works.
